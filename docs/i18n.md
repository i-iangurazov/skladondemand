# i18n

Setup
- Uses `next-intl` with JSON messages in `apps/web/src/messages/{en,ru,kg}.json`.
- Locale is resolved from the `NEXT_LOCALE` cookie (set by `useLanguage`), with `en` as fallback.
- Dev-only guard: `validateMessages()` compares `en` keys against `ru` and `kg` and throws if any are missing.
- Missing keys log warnings via `getMessageFallback` in `apps/web/src/app/layout.tsx`.

How to add strings
1) Add the new key to `apps/web/src/messages/en.json`.
2) Add the same key to `apps/web/src/messages/ru.json` and `apps/web/src/messages/kg.json`.
3) Use the key in code with `useTranslations()` (client) or `getTranslations()` (server).

Usage
Client components:
```tsx
import { useTranslations } from 'next-intl';

const t = useTranslations();
return <button>{t('common.actions.save')}</button>;
```

Server components:
```tsx
import { getTranslations } from 'next-intl/server';

const t = await getTranslations();
return <h1>{t('owner.analytics.title')}</h1>;
```

Namespace example (server):
```tsx
const t = await getTranslations('home');
return <h1>{t('headline')}</h1>;
```

Key conventions
- `common.*` for shared actions/labels/states.
- Page/role-specific namespaces: `guest.*`, `kitchen.*`, `staff.*`, `admin.*`, `owner.*`.
- Form placeholders/validation: `forms.placeholders.*`, `forms.validation.*`.
- Statuses: `status.*`.
